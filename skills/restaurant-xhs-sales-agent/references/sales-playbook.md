# Sales Playbook

## Priority

This skill is sales-first. Most outputs should be ready-to-send WeChat wording.

## Qualification Questions

Ask for:

1. 品牌名称
2. 所在城市
3. 门店数量
4. 品类和人均
5. 月营业额或大概推广预算
6. 小红书/抖音/点评现状
7. 是否有团购、点评、地图、门店承接
8. 当前最想解决的问题

## Opening Message

```text
老板好，我刚看了下你们这个城市和品类。

你们这种餐饮，不是只适合做普通探店，更适合做“城市美食攻略/旅游攻略”的铺量。

因为顾客不会一上来搜你的店名，他会先搜：
XX美食推荐、XX旅游吃什么、XX本地人推荐、XX不踩雷餐厅。

如果这些入口里看不到你，顾客做选择的时候就很容易被同行截走。

你把品牌名、城市、门店数量、人均、主打菜和现在的小红书账号发我，我先帮你判断适不适合做攻略铺量。
```

## First Diagnosis Reply

```text
我先不急着跟你推荐套餐。

这个业务要先判断三个点：

第一，你所在城市有没有用户搜索美食/旅游攻略的需求；
第二，你这个品类有没有适合被放进攻略的理由；
第三，你现在的线上内容能不能接住顾客的搜索和比较。

这三个点成立，攻略铺量才有价值。
如果不成立，发再多内容也只是热闹。
```

## Explain The Business

```text
我们做的不是传统代运营，也不是单纯找达人探店。

攻略铺量的核心，是把品牌放进顾客会搜索、会收藏、会比较的内容路径里。

比如用户搜“城市+美食推荐”“城市+旅游攻略”“本地人推荐餐厅”的时候，能不能看到你。

看到之后，内容够不够多、够不够真实、够不够让他愿意收藏和到店。
```

## Recommend 12k Package

```text
如果你们想认真做，我建议直接看1.2万的优选达人套餐。

这个套餐不是最低价，但更适合对内容质量有要求的餐饮品牌。

它包含美食攻略和旅游攻略两类内容，覆盖小红书、抖音、携程、大众点评，重点是帮你们在用户做攻略和搜索时增加出现频次。

轻量套餐也能做，但更像基础测试；如果你们品牌本身有一定体量，我不建议只为了省预算做最低配。
```

## Objection: Too Expensive

```text
我理解，预算肯定要算。

但这个项目不能只按“几篇内容多少钱”来看。

你要看的是：顾客搜城市美食、旅游攻略、品类推荐的时候，你有没有机会出现。

如果一个月营业额已经到几十万，真正浪费的不是这1万多预算，而是高意向顾客每天在小红书做选择时都看不到你。

如果你只是想低成本测试，可以做9500轻量套餐。
但如果你希望内容质量和资源更稳，我还是建议1.2万优选达人套餐。
```

## Objection: Can You Guarantee Results?

```text
这个我不能跟你乱保证。

小红书的排名、到店和转化，会受到关键词竞争、内容质量、账号权重、团购页面、点评评价、门店位置和服务承接影响。

我们能保证的是执行动作：
关键词规划、攻略内容制作、账号组合、平台分发、发布节奏和复盘。

专业做法不是承诺“必爆”或“必到店”，而是把能影响顾客决策的内容入口先铺起来。
```

## Objection: We Already Have Douyin/Dianping

```text
抖音和点评当然要做，但它们解决的问题不完全一样。

点评更像用户临门一脚确认，抖音更偏兴趣推荐和团购转化。

小红书攻略的价值，是在用户做计划、做搜索、做收藏的时候提前影响他。

尤其是餐饮和旅游城市，很多人是先在小红书做攻略，再去点评看评价和套餐。

所以不是替代，而是补齐决策前端。
```

## Objection: We Only Want One Or Two Posts

```text
一两篇不是不能做，但它验证不出攻略铺量的价值。

攻略业务看的是关键词覆盖、内容数量、账号组合和持续出现。

如果只发一两篇，很容易变成零散曝光，看不出搜索占位，也看不出用户收藏和比较的效果。

如果预算有限，可以先做轻量套餐，但不建议只拿一两篇来判断整个业务有没有用。
```

## Close

```text
这样，我建议你先把品牌信息发我：

品牌名、城市、门店数、人均、主打菜、当前小红书账号、预算范围。

我先给你判断适合做9500轻量、1.2万优选，还是1.665万报备投流。

不适合我也直接说，避免你花冤枉钱。
```
